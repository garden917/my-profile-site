화면 설계 (Wireframes)

3가지 핵심 기능이 실제로 어떤 화면으로 구현되는지 — 보이스봇 관리, 상담원 지원, 고객사 온보딩

🎨 3개 핵심 화면 · ⏱ 읽기 시간 약 10분 · 👤 민정원
01 설계 개요

3개 화면, 3개의 사용자

각 화면은 완전히 다른 역할의 사용자를 위해 설계되었다. 같은 플랫폼이지만 접근하는 사람이 다르면 화면이 달라야 한다.

🎛️
보이스봇 관리 콘솔
운영자 · 전문가용
CX팀 리더 또는 IT 운영팀이 봇 설정·성과 모니터링을 한 곳에서 처리. 기술 담당자 없이 운영 가능한 복잡도 설계.
🎧
Agent Assist 패널
상담원 · 현장용
고객과 통화 중인 상담원이 실시간으로 정보를 받는 화면. 응대 흐름을 방해하지 않는 낮은 인지 부하로 설계.
🚀
온보딩 위저드
고객사 담당자 · 비기술용
신규 고객이 IT 도움 없이 3일 내 스스로 봇을 세팅하고 실운영까지 완료. 최소 입력으로 최대 효과.
설계 원칙

이 3개 화면은 각각 "전문가용 (복잡도 허용)", "현장 실무자용 (응답 속도 최우선)", "비기술 최종 사용자용 (학습 곡선 최소화)"의 UX 목표를 가진다. 고객사의 규모나 역할에 관계없이 필요한 사람이 필요한 기능에 마찰 없이 접근할 수 있도록 설계했다.

02 보이스봇 관리 콘솔

운영자가 봇을 완전히 통제하는 화면

고객사 CX팀 리더나 IT 운영팀이 보이스봇 설정, 성과 모니터링, 지식 관리를 한 곳에서 처리한다.

Admin Dashboard
보이스봇 운영 관리 콘솔
주 사용자: 박지현 (CS센터장) 같은 운영 관리자. 상담원 120명을 관리하며 KPI를 매일 확인해야 하는 역할.
💡 설계 질문: "보이스봇을 만들고 운영하는 모든 것을 한 곳에서 관리할 수 있을까? IT팀 티켓 없이 운영자 혼자서?"
핵심 기능 (탭 구조)
  • 대시보드: 오늘의 KPI — 통화수, 자기서비스율, AHT, 오류율 실시간 표시
  • 봇 설정: 페르소나(이름·인사말·톤), 에스컬레이션 규칙 변경
  • 지식 관리: PDF/URL 업로드 → 자동 임베딩 → 처리 상태 추적
  • 통화 로그: 녹음 재생, 감정 점수, 자기서비스 여부 필터·검색
  • LLMOps: 프롬프트 버전 관리, A/B 테스트 결과 시각화
기대 효과
운영자가 기술 담당자 없이 봇 설정을 수정하고 성과를 추적할 수 있어 IT 의존도 ↓, 운영 반응속도 ↑. 이슈 발생 시 대시보드에서 즉각 감지 가능.

03 Agent Assist 패널

통화 중 상담원의 두 번째 눈

상담원이 고객과 대화하는 동안 실시간으로 정보 추천·감정 변화 알림·자동 요약을 제공하는 화면. 화면을 보면서도 통화에 집중할 수 있도록 정보 밀도를 제어했다.

Agent Assist · Live Call
실시간 상담원 지원 화면
주 사용자: 이수진 (수석 상담원) 같은 현장 상담원. 하루 80통화를 처리하며 복잡한 금융 상품 문의 시 3~4개 탭을 동시에 조회해야 했던 상황을 해결.
💡 설계 질문: "고객 정보와 필요한 정책을 한 화면에서 볼 수 있다면, 탭 전환 시간이 사라지고 AHT가 줄어들지 않을까?"
레이아웃: 3단 분할 구조
  • 좌측 25%: 고객 정보 패널 — 이름·VIP 등급·계정 이력·진행 중 이슈
  • 중앙 50%: 실시간 트랜스크립트 + 감정 인디케이터 (색상 바)
  • 우측 25%: 동적 배틀카드 — 고객 발화에 따라 자동 추천되는 정책·FAQ
통화 종료 후 자동화
  • 통화 내용 2초 내 자동 요약 생성
  • CRM 저장 버튼 한 번으로 완료
  • 수기 작업 완전 제거
기대 효과
신입도 숙련 수준의 응답 가능. AHT -23%, FCR 70%+, 통화 후 수기 작업 제거. 상담원 번아웃 감소로 이탈률 개선 기대.

04 온보딩 위저드

SI 6개월을 3일로

신규 고객사 담당자가 IT 도움 없이 가입부터 실운영까지 4단계로 완료하는 셀프서비스 플로우. TTF(Time to First Value) 3일 달성의 핵심 화면이다.

Onboarding Wizard
고객사 셀프 온보딩 위저드
주 사용자: 김민수 (이커머스 CS팀장) 같은 고객사 담당자. "SI 도입엔 6개월이 걸려서 이번 시즌을 맞출 수 없다"는 문제를 해결하기 위한 화면.
💡 설계 질문: "기술 담당자 없이도 이번 주 안에 보이스봇을 실운영할 수 있을까? 4번의 입력으로?"
4단계 플로우
  • Step 1: 회사 정보 — 산업·규모·월 통화량 입력 (봇 초기 설정 자동 최적화)
  • Step 2: 봇 페르소나 — 이름·환영 메시지·말투 설정
  • Step 3: 지식 업로드 — PDF/URL 업로드 → 자동 임베딩 처리
  • Step 4: 샘플 테스트 — 실제 통화 시뮬레이션 후 실운영 전환
UX 원칙
  • 필수 입력만 노출, 선택 항목은 "고급 설정"으로 숨김
  • 실시간 유효성 검사로 오류 즉시 안내
  • 진행률 표시로 "다 왔다"는 심리적 안도감
기대 효과
TTF 3일 달성. 이커머스 담당자가 시즌 2주 전에 혼자 세팅 완료. 판매 주기 75% 단축, 초기 레퍼런스 확보 가속화.
