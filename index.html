📌 과제 개요

무엇을 기획했는지, 어떤 자산에서 출발했는지

배경과 문제 인식

뤼튼은 이미 금융·보험 업종에 구축형(SI) AICC 솔루션을 납품한 경험이 있다. 구축형은 고객사별 커스터마이징이 수반되어 운영 비용이 누적되고, 계약 종료 후 수익이 끊기는 구조다. 이 과제는 그 기술 자산을 재활용하여 구독 기반 멀티테넌트 SaaS로 전환하는 제품 전략을 수립하는 것이다. 고객이 늘수록 단위 비용이 줄고 NRR(Net Revenue Retention)이 쌓이는 수익 구조를 목표로 한다.

재활용 가능한 기존 자산

  • ·보이스봇 코어 엔진 — 도메인별 학습된 STT/TTS 파이프라인
  • ·Elasticsearch + Neo4j RAG — 지식베이스 검색·추론 인프라
  • ·React AdminPanel — 관리자 콘솔 UI 기반

타겟 시장

  • ·이커머스 — 스파이크성 CS 볼륨, 빠른 의사결정 (초기 타겟)
  • ·금융·보험 — 규제 대응 필요, 검증 후 진입 (Phase 2+)
  • ·제조·전자 — 기술 고객 대응, 장기 확장 영역

4개 산출물

  • ·제품 기획서 (PRD)
  • ·화면 설계 (Wireframes)
  • ·제품 로드맵 (10개월)
  • ·가격 전략 (3-Tier + 성과기반)

🎯 핵심 의사결정 5가지

이 기획 전체를 관통하는 전략적 판단들. 각 결정이 왜 그 방향이어야 하는지 근거를 함께 확인할 수 있다.

세부 논거는 각 산출물 문서에서 확인 가능

결정 항목 선택안 비교 방향 판단 근거
제품 포지셔닝 SI 유지 vs SaaS 전환 SaaS 중심 SI는 프로젝트 단위 수익으로 고객 수가 늘수록 오히려 운영 비용이 비례 증가한다. SaaS 전환 시 멀티테넌트 공유 인프라 위에서 한계비용이 줄고, 구독 갱신을 통한 NRR 누적이 가능해진다. 기존 코어 엔진을 그대로 재활용할 수 있어 전환 비용도 최소화된다.
초기 타겟 금융 우선 vs 이커머스 우선 이커머스 우선 금융·보험은 규제 대응(망분리, 개인정보보호법 등) 요구로 온보딩에 3~6개월이 소요되어 초기 레퍼런스 확보가 어렵다. 이커머스는 시즌 스파이크 CS 볼륨 문제가 명확하고 의사결정이 빠르다. 이커머스 레퍼런스를 먼저 확보한 후 금융권 영업에 활용하는 순서가 현실적이다.
신규 개발 우선순위 Agent Assist vs 기타 기능 Agent Assist 보이스봇은 기존 자산으로 이미 보유 중이므로 신규 개발 리소스를 투입할 이유가 없다. Agent Assist는 상담원이 고객 응대 중에 실시간 답변 추천·지식베이스 검색을 받는 기능으로, 도입 즉시 AHT 단축 효과가 수치로 나타난다. 이 ROI가 Business 플랜 업셀 근거가 된다.
멀티테넌시 아키텍처 공유 스키마 vs 전용 DB 하이브리드 완전 공유 스키마는 인프라 비용이 낮지만, 금융·보험 고객은 데이터 물리 분리를 계약 조건으로 요구하는 경우가 많다. 완전 전용 DB는 고객당 비용이 과다하다. SMB 고객은 공유 스키마, Enterprise 고객은 전용 DB를 티어별로 선택 가능하게 하는 하이브리드 구조가 비용과 보안을 동시에 만족시킨다.
가격 모델 정액제 vs 성과 기반 단계적 전환 성과 기반 과금(AI 해결률·AHT 기준)은 고객 입장에서 리스크 없는 구조지만, 도입 초기엔 측정 기준에 대한 신뢰가 없어 계약 마찰이 생긴다. Phase 1~2는 정액으로 신뢰와 데이터를 쌓고, Phase 2 후반부터 성과 기반을 옵션으로 추가하는 것이 현실적인 전환 경로다.

📑 산출물 목록 & 추천 읽기 순서

각 문서가 어떤 질문에 답하는지 먼저 파악하면 더 빠르게 검토할 수 있다. 총 정독 기준 약 50분.

1 제품 기획서 (PRD) 📄 6개 섹션 · 15분

답하는 질문: 이 제품은 무엇이고, 누구를 위해 무엇을 해결하는가?
제품 비전과 포지셔닝, 세그먼트별 타겟 고객 페르소나, 기존 자산을 어떻게 활용하는지, 기능 우선순위와 Phase별 KPI 목표까지 담겨 있다. 전체 기획의 "왜"에 해당하는 문서로, 나머지 산출물을 이해하는 데 필요한 배경이 모두 여기 있다.

2 화면 설계 (Wireframes) 🎨 3개 화면 · 10분

답하는 질문: 실제로 어떤 화면으로 구현되는가?
보이스봇 관리 콘솔(운영자가 봇 시나리오를 설정하는 화면), Agent Assist 패널(상담원이 실시간으로 AI 추천을 받는 화면), 고객사 온보딩 위저드(신규 고객이 셀프로 세팅을 완료하는 4단계 플로우)를 담았다. PRD의 기능 기술을 UI로 구체화한 문서다.

3 제품 로드맵 📊 10개월 계획 · 12분

답하는 질문: 언제, 무엇을, 어떤 순서로 만드는가?
Phase 0(기반 정비 2개월) → Phase 1(MVP 출시 1개월) → Phase 2(기능 확장 3개월) → Phase 3(성숙화 4개월)의 4단계 계획을 담았다. 기능 간 의존성 맵, 팀 구성 및 리소스 할당, 주요 리스크와 대응 방안까지 포함한다. "실행 가능한가"를 검토하기 위한 문서다.

4 가격 전략 💰 3가지 모델 · 10분

답하는 질문: 어떻게 돈을 버는가, 가격은 왜 이 수준인가?
Starter(150만원/월) · Business(500만원/월) · Enterprise(협의)의 3-Tier 정액 모델과, Phase 2 이후 도입할 성과 기반 과금(AI 해결 건당 과금)을 다룬다. 경쟁사 벤치마크와 고객 가치 기준 가격 근거도 포함되어 있어, 가격이 임의로 설정된 것이 아님을 확인할 수 있다.

⚡ 주요 수치 참조

문서를 읽다가 수치의 맥락이 필요할 때 참고. 각 수치가 왜 이 목표인지 근거는 PRD 및 Pricing 문서에 상세히 기술했다.

🎯 성과 KPI 목표
  • 자기서비스율 60%+
    업계 평균 35% 대비 목표치. 달성 시 상담 인력 20~30% 절감 효과
  • AHT -23% 이상
    Agent Assist 도입 파일롯 기준. 상담당 처리시간 단축
  • FCR 70%+
    첫 접촉 해결률. 재문의 감소 → 전체 CS 볼륨 압축
  • 온보딩 TTF 3일
    Time to First Value. 셀프서비스 온보딩 위저드 기준
💰 가격 구조
  • Starter — 150만원/월
    보이스봇 기본 + 5석 이하 SMB. 이커머스 초기 진입 타겟
  • Business — 500만원/월
    Agent Assist + 무제한 석. AHT 절감 ROI로 가격 정당화
  • Enterprise — 협의
    전용 DB + SLA + 커스텀 연동. 금융·보험 메인 타겟
  • 성과기반 — 기본료 + 건당
    Phase 2 이후 옵션. AI 해결 건당 2,000~5,000원 구조
📅 개발 타임라인
  • Phase 0 — M1~M2 (2개월)
    멀티테넌트 기반 구축, 기존 엔진 SaaS화, 인프라 세팅
  • Phase 1 — M3 (1개월)
    MVP 출시. 보이스봇 SaaS + 관리 콘솔 + 온보딩 위저드
  • Phase 2 — M4~M6 (3개월)
    Agent Assist + 감정분석 + Business 플랜 출시
  • Phase 3 — M7~M10 (4개월)
    성과기반 과금, Enterprise 기능, 글로벌 확장 준비
