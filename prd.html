01 Executive Summary

"Bring AI to Every Call"

이 제품이 해결하는 문제, 우리가 선택한 방향, 그리고 왜 지금인가

제품 정의

금융·보험·이커머스 콜센터의 인바운드·아웃바운드 통화를 AI가 자동 처리하거나 상담원을 실시간으로 보조하는 범용 멀티테넌트 SaaS 플랫폼. 고객사는 3일 이내 셀프 온보딩으로 즉시 사용 가능하며, 자기서비스율 60%+, AHT -23%, 통화당 비용 -40%의 효과를 목표로 한다.

현재 상황과 전환의 논리
SI 모델의 한계
  • 고객사별 커스터마이징으로 운영 비용 누적
  • 프로젝트 종료 후 수익 단절 (일회성 구조)
  • 온보딩에 3~6개월 소요, 빠른 확장 불가
  • 고객 수 증가가 오히려 비용 증가로 이어짐
SaaS 전환의 이점
  • MRR 기반 예측 가능한 수익 구조
  • 고객 수 증가 → 단위 비용 감소 (공유 인프라)
  • 셀프 온보딩으로 TTF 3일 달성
  • NRR 갱신·업셀을 통한 지속 성장 가능

핵심 자산: 이미 금융·보험사 납품을 통해 검증된 보이스봇 코어 엔진, Elasticsearch + Neo4j RAG 인프라, React AdminPanel이 존재한다. 이 기획은 0→1이 아닌 기존 자산의 제품화(productization)로, 신규 개발 범위를 최소화하면서 SaaS 전환이 가능하다.

02 시장 분석

왜 지금인가?

시장의 구조적 문제와 우리가 진입할 수 있는 이유

고객사들이 공통적으로 겪는 문제
비용 구조

상담원 1명/월 200만원 이상, 연간 이탈률 30%+로 채용·교육 비용 반복 발생. 스파이크 시즌엔 임시 인력 수급까지 겹쳐 비용이 연간 수억 단위로 증가한다.

품질·합규 리스크

통화 모니터링·합규성 검수가 수동 프로세스에 의존하고 있어 음성·개인정보 규제 강화 시 즉각 대응이 어렵다. 사후 적발 시 법적 리스크로 이어진다.

기존 솔루션의 한계

Genesys·Vonage 등 글로벌 솔루션은 가격이 agent당 $75~150/월 수준이고 한국어 특화가 부족하다. 국내 SI 업체는 도입에 6개월+가 걸린다.

타이밍 압박

LLM 발전으로 AI 음성 응대가 상용 수준에 도달했고, 경쟁사들이 빠르게 진입 중이다. 지금이 초기 레퍼런스 확보의 마지막 적기다.

시장 규모 — TAM / SAM / SOM

국내 컨택센터 산업 통계(고용부 2023)와 경쟁사 가격 데이터를 기반으로 역산한 추정치다. TAM은 AI 솔루션으로 대체·효율화 가능한 인건비 총량 기준이며, 실제 시장 침투는 SOM 범위에서 시작한다.

03 솔루션

우리는 무엇을 만드는가?

3가지 핵심 제품으로 구성된다. 보이스봇은 기존 자산 기반, Agent Assist와 온보딩 위저드는 신규 개발.

🤖
제품 A · 기존 자산 기반
AI 보이스봇 (SaaS 래핑)

고객 음성을 실시간으로 처리해 자동 응답하거나 상담원에게 전달하는 인바운드·아웃바운드 봇. 기존 코어 엔진을 SaaS 멀티테넌시 구조로 감싸고, 고객사별 API 과금을 붙이는 것이 핵심 작업이다.

핵심 기능
  • STT → LLM → TTS 파이프라인 (응답 지연 1.5~1.8초)
  • 하이브리드 RAG 기반 정책 정확 인용
  • Closed-loop 태스크 자동 실행 (주소변경, 환불, 조회)
  • 부정 감정 감지 시 자동 상담원 전환
기대 효과
  • 자기서비스율 60%+ (업계 평균 35% 대비)
  • 통화당 처리 비용 -40%
  • 24/7 무중단 대응 (스파이크 무관)
👥
제품 B · 완전 신규 개발
Agent Assist (상담원 AI 코파일럿)

상담원이 고객과 통화하는 동안 실시간으로 정보를 추천하고 감정 변화를 알려주는 화면. 기존 보이스봇이 처리 못하는 복합·감정적 문의를 커버하면서 AHT를 단축시키는 핵심 프리미엄 기능이다. Business 플랜 업셀의 핵심 근거이며, 이 기능의 ROI가 수치로 바로 나타난다는 점에서 신규 개발 우선순위 1위다.

핵심 기능
  • 실시간 트랜스크립트 & 감정 분석
  • 동적 배틀카드 (고객 질문에 최적 정책 자동 추천)
  • 통화 후 자동 요약 → CRM 자동 저장
  • 고객 정보 패널 (계정 이력, 진행 중 이슈)
기대 효과
  • AHT 23% 단축 (파일롯 기준)
  • 신입 온보딩 기간 50% 감소
  • 통화 후 수기 작업 제거 (상담원 만족도 개선)
🚀
제품 C · 완전 신규 개발
고객사 셀프 온보딩 위저드

고객이 가입부터 봇 설정, 실운영까지 3~5일 내 자체적으로 완료할 수 있는 4단계 가이드 UI. SaaS 모델의 TTF(Time to First Value) 단축이 핵심이고, 이 온보딩 경험의 완성도가 초기 레퍼런스 확보와 직결된다.

05 구현 전략

어떻게 만드는가?

기존 자산 활용 범위, 4단계 개발 계획, 필요 리소스

기존 자산 활용 vs 신규 개발

이미 검증된 자산이 전체의 약 60%. 나머지 40%만 신규 개발하면 SaaS 전환이 가능하다는 것이 이 기획의 핵심 전제다.

자산 현 상태 SaaS 전환 작업 개발 공수
보이스봇 코어 엔진 ✓ 검증됨 멀티테넌시 격리 레이어 추가, 호출 API 표준화 경미
Elasticsearch + Neo4j RAG ✓ 프로덕션 테넌트별 인덱스 격리 (DLS 정책 적용) 경미
React AdminPanel ✓ 기존 UI 다중 고객사 테넌트 대응 구조로 개선 중간
Agent Assist 화면 ✗ 신규 실시간 트랜스크립트·추천·감정 분석 UI 전체 설계 완전 신규
온보딩 위저드 ✗ 신규 4단계 셀프서비스 플로우, 지식 업로드 파이프라인 완전 신규
멀티테넌시 보안 ✗ 미흡 JWT + RBAC, 테넌트 간 데이터 격리 아키텍처 재설계 아키텍처 재설계
4단계 배포 계획 (총 10개월)

Phase 0이 완전히 끝나야 Phase 1이 시작 가능하다. 멀티테넌시 기반이 없는 상태에서 MVP 출시는 보안 사고 리스크가 있다.

Phase 0 · M1~M2 · 2개월
기반 정비 — SaaS 출시 가능한 인프라 완성

이 단계를 건너뛰면 고객사 데이터가 섞이는 치명적 보안 사고 가능성이 있다. 빠른 MVP 욕심보다 기반 완성이 우선이다.

  • Elasticsearch DLS 정책으로 테넌트별 인덱스 격리
  • JWT + RBAC 기반 권한 체계 설계
  • 과금 미터링 기초 인프라 구축
Phase 1 · M3 · 1개월
MVP 출시 — 첫 고객 온보딩 & 실운영

이커머스 1~2개 고객사 파일롯 운영 시작. 이 시점의 TTF와 자기서비스율이 이후 영업 레퍼런스가 된다.

  • 보이스봇 SaaS 래핑 완료 (호출 API 표준화, 과금 연동)
  • Agent Assist 기본 버전 (트랜스크립트 + 지식 추천)
  • 온보딩 위저드 4단계 + 샘플 통화 테스트
Phase 2 · M4~M6 · 3개월
기능 확장 — Business 플랜 출시 & 고객 만족도 증대

파일롯 고객 피드백을 반영하고, AHT 절감 효과가 수치로 확인되면 Business 플랜 영업 레버로 활용한다.

  • 감정 분석 + 자동 에스컬레이션 고도화
  • CRM 연동 (Salesforce, HubSpot 기본 API)
  • LLMOps 대시보드 (프롬프트 A/B 테스트, 성능 모니터링)
Phase 3 · M7~M10 · 4개월
성숙화 — 경쟁 우위 확보 & 성과 기반 과금 전환

누적 데이터를 기반으로 성과 측정 기준이 확립된 고객사에 한해 성과 기반 과금 옵션을 제공하고, 금융권 Enterprise 진입을 본격화한다.

  • 도메인별 특화 모델 파인튜닝 (금융·이커머스·보험)
  • 성과 기반 과금 미터링 (Resolved Call 정의 및 추적)
  • Enterprise 기능: 전용 DB 옵션, SLA 계약, 커스텀 연동
필요 리소스

총 6명 규모. MLOps 인재가 채용 시장에서 가장 빠듯하므로 조기 채용 시작이 필요하다.

역할 인원 주요 책임 주요 Phase
Backend Engineer 2명 멀티테넌시 아키텍처, API 표준화, 과금 미터링 Phase 0~1 집중
Frontend Engineer 2명 Agent Assist UI, 온보딩 위저드, AdminPanel 개선 Phase 1~2 집중
