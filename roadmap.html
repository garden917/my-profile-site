02 Phase 0 · M1~M2

기반 정비 — SaaS 출시 가능한 상태 만들기

이 단계를 건너뛰면 고객사 데이터가 혼재되는 치명적 보안 사고가 발생할 수 있다. 빠른 출시보다 기반 완성이 절대적 우선순위다.

Phase 0 · 기반 정비 · 2개월
핵심 목표: 다수의 고객사가 동시에 플랫폼을 사용하더라도 서로의 데이터를 절대 볼 수 없는 보안 격리 아키텍처 완성. 이 없이는 Phase 1 착수 불가.
실행 항목
  • 멀티테넌시 데이터 격리 — Elasticsearch DLS(Document-Level Security) 구현, 테넌트 간 인덱스 격리
  • JWT 인증 체계 — 고객사별 토큰 발급·검증 로직
  • RBAC 권한 관리 — 역할별 접근 제어 (관리자·상담원·감독자)
  • 감시 통화 합법성 — 녹음 동의·개인정보 처리 동의 프로세스
  • 과금 미터링 기초 — 통화 단위 집계 인프라
팀 구성 (6명)
  • Backend 2명 — 아키텍처 설계, Elasticsearch DLS
  • Security 1명 (외부 또는 겸직) — 침투 테스트, 보안 감사
  • DevOps 1명 — 인프라 구성, CI/CD
  • Frontend 1명 — AdminPanel 기본 틀
  • PM 1명 — 요구사항 정의, 의존성 관리
Phase 0 완료 기준 (이것이 되어야 Phase 1 착수)
  • 고객사 A 데이터가 고객사 B에게 노출되지 않음 (침투 테스트 통과)
  • SOC 2 / ISO 27001 준비 문서 초안 완성
  • 통화 1건당 과금 단위 집계 정상 동작
03 Phase 1 · M3

MVP — 첫 고객사 실운영 시작

"3일 온보딩"을 실제로 입증하는 단계. 이커머스 파일롯 고객 1~2개사를 통해 가설을 검증하고 영업 레퍼런스를 확보한다.

Phase 1 · MVP · 1개월
핵심 목표: TTF(Time to First Value) 3일 달성을 파일롯 고객사 2개 이상에서 입증. 이 숫자가 이후 영업의 핵심 메시지가 된다.
실행 항목
  • 보이스봇 SaaS 래핑 — 기존 엔진에 API 표준화, 호출 단위 과금 연동
  • Agent Assist 기본 — 실시간 트랜스크립트 + 지식 검색 자동 추천
  • 온보딩 위저드 4단계 — 셀프서비스로 3일 내 세팅 완료
  • 관리 콘솔 기본 — KPI 대시보드, 봇 설정, 통화 로그
  • 24/7 기술 지원팀 — 파일롯 기간 긴급 대응
팀 구성 (5명)
  • Backend 2명 — API 표준화, 과금 연동
  • Frontend 2명 — Agent Assist, 온보딩 위저드
  • PM 1명 — 파일롯 고객 관리, 피드백 수집
Phase 1 성공 기준
  • 파일롯 고객사 2개 이상 — 온보딩 3일 이내 완료
  • 자기서비스율 60%+ 달성 (파일롯 2주 운영 기준)
  • TTF 3일 달성 → 마케팅 자료 및 영업 레퍼런스로 활용
  • SI 대비 비용 80% 절감 수치 산출
04 Phase 2 · M4~M6

기능 확장 — Business 플랜과 수익 가속화

파일롯 피드백을 반영하고, AHT 절감 수치가 나오면 Business 플랜(500만원/월)의 영업 레버로 활용한다. 고객 6개사, MRR 2천만원이 이 단계의 목표다.

Phase 2 · 기능 확장 · 3개월
핵심 목표: Agent Assist를 통한 AHT -23% 달성을 수치로 증명하고, 이를 Business 플랜 업셀의 근거로 활용. 고객 6개사 확보, MRR 2천만원 돌파.
실행 항목
  • 감정 분석 & 에스컬레이션 자동화 — 실시간 감정 점수, 부정 임계값 자동 상담원 전환
  • 통화 자동 요약 + CRM 연동 — Salesforce·HubSpot 기본 API 연동
  • LLMOps 대시보드 — 프롬프트 A/B 테스트, 응답 품질 추적
  • 아웃바운드 캠페인 — 보이스봇 기반 자동 발신 (프로모션·설문)
  • 고급 RBAC — 팀·부서별 권한 분리
팀 구성 (6명)
  • Backend 2명 — CRM 연동, 아웃바운드 로직
  • Frontend 1명 — LLMOps 대시보드
  • MLOps 1명 — 감정 분석 모델, 프롬프트 최적화
  • QA 1명 — 감정 분석 정확도 검증
  • PM 1명 — Business 플랜 출시 관리
Phase 2 성공 기준
  • AHT -23% 달성 (파일롯 고객사 Before/After 비교)
  • 고객사 6개사 이상 확보 (Starter 2 + Business 4 기준, 500만×4+150만×2=2,300만)
  • MRR 2천만원+ 돌파
  • NRR 120% 달성 — 기존 고객의 플랜 업그레이드로 확인
05 Phase 3 · M7~M10

성숙화 — 성과 기반 과금과 경쟁 우위 확보

누적된 데이터를 기반으로 성과 측정 기준이 확립된 고객에게 성과 기반 과금 옵션을 제공하고, 금융권 Enterprise 진입을 본격화한다.

Phase 3 · 성숙화 · 4개월
핵심 목표: "우리는 고객의 성과에만 돈을 받는다" — 성과 기반 과금 도입으로 신뢰·점유율 극대화. MRR 1억원, 고객 20개사+.
실행 항목
  • 도메인별 특화 LLM — 금융·이커머스·보험 도메인 파인튜닝
  • 성과 기반 과금 미터링 — "해결된 통화" 판정 로직, 투명한 청구 대시보드
  • Enterprise 기능 — 전용 DB 옵션, SLA 계약, 커스텀 연동
  • 마켓플레이스 — 3rd-party CRM 플러그인, 파트너 API
  • AI 모니터링 — 편향 감지, 규정 위반 자동 알림
팀 구성 (8명)
  • Backend 2명 — Enterprise 기능, 미터링 인프라
  • Frontend 1명 — 성과 대시보드, 마켓플레이스
  • MLOps 2명 — 도메인 특화 모델 파인튜닝
  • Sales/CS 1명 — Enterprise 영업, 금융권 대응
  • PM 1명 + QA 1명
Phase 3 성공 기준
  • MRR 1억원 달성 (Series A 펀딩 준비 트리거)
  • 고객 이탈률 2% 이하 (성과 기반 신뢰 반영)
  • 성과 기반 과금 고객사 5개 이상 (전환 동의 기준)
  • 금융·보험 Enterprise 고객 2개사 이상 확보
06 의존성 & 리스크

기능 의존성 맵 & 주요 리스크

의존성을 무시한 병렬 개발은 재작업과 통합 지연을 유발한다. 각 기능의 선행 조건을 확인하고, 착수 순서를 지켜야 한다.

기능 의존성 맵

각 기능은 선행 기능이 완료된 이후에만 착수 가능하다. Phase 0 → 1 → 2 → 3 순서는 협의 불가한 강제 순서다.

기능 Phase 선행 의존 기능 이유
멀티테넌시 데이터 격리 Phase 0 없음 (최상위 선행) 모든 SaaS 기능의 기반. 없으면 고객사 데이터 혼재 위험.
JWT 인증 / RBAC Phase 0 없음 (최상위 선행) 멀티테넌시와 병렬 개발 가능. 단, Phase 1 전 반드시 완료.
보이스봇 SaaS 래핑 Phase 1 멀티테넌시 격리 API 호출 단위 과금은 테넌트 식별이 선행되어야 설정 가능.
온보딩 위저드 Phase 1 JWT 인증 / RBAC 계정 생성·권한 부여 로직이 완성되어야 위저드 진행 가능.
Agent Assist 기본 Phase 1 보이스봇 SaaS 래핑 실시간 트랜스크립트 스트림은 보이스봇 API 표준화 이후 연결 가능.
감정 분석 & 에스컬레이션 Phase 2 Agent Assist 기본 트랜스크립트 스트림 위에 감정 모델을 올리는 구조. 기반 없이 독립 개발 불가.
통화 요약 + CRM 연동 Phase 2 Agent Assist 기본 요약 생성은 트랜스크립트 데이터 기반. CRM 연동은 요약 포맷 확정 후 가능.
성과 기반 과금 미터링 Phase 3 보이스봇 API + 감정 분석 "해결된 통화" 판정은 에스컬레이션 여부(감정 분석)와 완결 로그(API) 둘 다 필요.
도메인별 특화 LLM Phase 3 LLMOps 대시보드 (Phase 2) A/B 테스트 결과로 파인튜닝 방향 결정. 충분한 트래픽 없이 만들면 과적합.
⚠️ 주의: Phase 1 착수 조건은 Phase 0 전체 완료. Phase 0가 지연되면 MVP 일정이 1:1로 밀린다. 멀티테넌시 격리와 RBAC는 병렬로 진행하되 Phase 0 내에 모두 완료해야 한다.
주요 리스크 & 대응
🔴 High Priority
멀티테넌시 보안 침해
고객 데이터 유출 발생 시 신뢰 추락과 법적 책임이 동시에 발생. 대응: 외부 보안 감사팀 고용, Phase 0 완료 후 월간 침투 테스트 정례화.
초기 고객 확보 지연
Phase 1 완료 후에도 실제 고객이 없으면 PMF 검증이 불가능하고 투자 리스크가 급증. 대응: Phase 0 시작 시점부터 이커머스 알파 고객 사전 확보, 무료 파일롯 제공.
감정 분석 정확도 문제
도메인별(금융·이커머스·보험)로 감정 표현 방식이 달라 공통 모델의 오분류율이 높을 수 있음. 대응: 고객사별 파인튜닝, 초기 3개월 수동 검수 병행.
🟡 Medium Priority
CRM 연동 복잡도 과소평가
Salesforce·HubSpot API는 버전 차이, 커스텀 필드 등 변수가 많아 일정 초과 위험. 대응: MVP에는 기본 API만 포함하고 Phase 2에서 점진적 확장. 연동 파트너사 활용 검토.
