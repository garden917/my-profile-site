제품 로드맵 & 실행 계획

AICC 범용 SaaS 플랫폼의 4단계 개발 계획 (10개월, M1~M10)

📊 전체 일정 및 리소스

Phase 기간 목표 리소스 예산 (추정)
Phase 0 M1~M2 (2개월) 기반 정비 (멀티테넌시, 보안) BE 2, MLOps 1 2억
Phase 1 (MVP) M3 (1개월) SaaS 베타 출시 FE 2, BE 2, MLOps 1, PM 1, BD 1 3억
Phase 2 M4~M6 (3개월) 고급 기능 + 고객 확장 FE 2, BE 2, MLOps 1, PM 1, BD 3 4억
Phase 3 M7~M10 (4개월) 성숙화 + 성과기반 과금 FE 2, BE 2, MLOps 1, Sales 추가 2억
합계: 10개월, 평균 6.5명, 개발 비용 ~11억원 (별도: GTM/마케팅 예산 1.5~10억원)
🏗️

Phase 0: 기반 정비

M1~M2 (2개월) | 목표: 멀티테넌시 아키텍처 완성

🔐 멀티테넌시 데이터 격리
MVP 보안 수준: Elasticsearch 인덱스 레벨 격리 (논리적 Dedicated), Neo4j 별도 DB, JWT 인증 강화 | DLS(Document Level Security) 구현은 Phase 2로 연기
⚙️ 보이스봇 코어 SaaS 래핑
기존 엔진 → API 기반 호출, 테넌트 식별자 전달, 통화 과금 미터링 구조
🔑 계정 시스템 & 권한 관리
사용자 인증 (OAuth2), 팀 기반 RBAC, API 키 관리
📊 기초 모니터링 & 로깅
통화 로그, 에러 추적, SLA 기초 구축
⚠️ 리스크: 멀티테넌시 보안 오류 (데이터 유출) → 검수 2회 필수 (내부, 외부 보안 감사)
🚀

Phase 1: MVP 출시 (Beta)

M3 (1개월) | 목표: 첫 고객사 온보딩 & 베타 운영

🎯 고객사 온보딩 위저드 (완전 신규)
4단계 셀프 서비스 / 회사정보 → 봇설정 → 지식업로드 → 테스트
🤖 Agent Assist 기본 기능 (완전 신규)
실시간 트랜스크립트 + 지식 자동 추천 (배틀카드) | 감정 분석은 Phase 2에서 추가
📊 보이스봇 관리 콘솔 개선
기존 AdminPanel → 대시보드 (KPI 카드), 통화 로그 조회
📈 기초 분석 & 보고
일일 통화량, 자기서비스율, AHT 기초 메트릭
✅ 성공 기준: 3개 이상 베타 고객사 가입, 월 1,000+통화, 자기서비스율 40% 이상
⚠️ 리스크: 초기 고객 획득 지연 → 마케팅팀 조기 투입 필요
⚡

Phase 2: 고도화 & 확장

M4~M6 (3개월) | 목표: 고급 기능 + 다수 고객 안정화

😊 감정 분석 + 에스컬레이션
실시간 감정 스코어링 (-1.0~1.0), 부정 임계값 기반 자동 전환 | 정확도 목표: 금융/이커머스별 80% 이상 (평균 75%+) | M4~M5 파인튜닝 + 2주 버퍼 확보
📝 자동 통화 요약 + CRM 연동
2초 내 요약 생성, Salesforce/HubSpot 기본 API 연동
📞 아웃바운드 캠페인 (추가 기능)
프로모션/설문 자동 전화, 콜 리스트 관리 | PRD에서 정의한 기본 기능 외 추가 범위
🧪 LLMOps 대시보드
프롬프트 A/B 테스트, 응답 품질 평가 (Eval), 모델 버전 관리
👥 팀 기반 권한 관리
부서별 역할 분리 (관리자, 분석가, 상담원)
📋 성과 기반 과금 감사 로그
해결 판정 근거 기록, 분쟁 해결 7~10일 SLA, 투명성 확보 | Phase 2 파일럿 운영 중 정확도 검증
✅성공 기준: 50+ 고객사, 월 200K+ 통화, 자기서비스율 55% 이상, NPS 45+
⚠️ 리스크: 감정분석 정확도 80% 미달 → 도메인별 모델 미세 조정 필요 (추가 3주) | 이 경우 Phase 3 일정 1개월 연기 가능성
🏆

Phase 3: 성숙화 & 경쟁력 강화

M7~M10 (4개월) | 목표: 도메인 특화 + 성과기반 과금

🎓 도메인별 특화 모델
금융 규정 특화, 이커머스 물류 최적화, 보험 청구 자동화
💰 성과 기반 과금 (Outcome-based)
해결된 통화(Resolved Call) 기준 미터링, 기본료 + 건당 과금
🏪 마켓플레이스 / API 생태계
CRM 플러그인, 써드파티 API 확장 (결제, 배송 등)
🛡️ AI 거버넌스 & 규정 준수
편향 감지, 개인정보 처리 모니터링, 규제 위반 알림
📱 모바일 앱 (선택사항)
iOS/Android 네이티브 앱, 모바일 상담원 지원
✅ 성공 기준 (M10 말 기준): 200+ 고객사, ARR $10M+, 자기서비스율 60%+, NPS 50+ | 월 15~20% 고객 성장률 달성 시 목표 달성
⚠️ 리스크: 성과기반 과금 미터링 오류 → 감사(Audit) 로그 필수

🚨 의존성 & 리스크 관리

🔗 크리티컬 의존성

  • Phase 0 → Phase 1: 멀티테넌시 보안 검수 완료 필수
  • Phase 1 → Phase 2: 초기 고객 3+사 온보딩 성공
  • Phase 2 → Phase 3: CRM 연동 API 표준화 완료
  • Phase 2 내 (M4~M5): 감정분석 모델 정확도 80% 이상 달성 (미달시 Phase 3 연기)

⚠️ 주요 리스크 & 대응

  • 기술: 멀티테넌시 성능 저하 → 로드 테스트 조기 실행
  • 시장: 초기 고객 확보 어려움 → 참조 고객(Ref) 기반 판매
  • 규제: 금융권 감시 강화 → 컴플라이언스팀 조기 투입
  • 인력: MLOps 인재 부족 → Phase 0부터 채용 시작

📍 주요 마일스톤

M2 말: 멀티테넌시 아키텍처 + 보안 검수 완료
M3 말: MVP 베타 출시 (3+ 고객사, 월 1K+ 통화)
M6 말: Phase 2 완료 (50+ 고객사, 자기서비스율 55%)
M10 말: Phase 3 완료 (200+ 고객사, 성과기반 과금 운영중)
